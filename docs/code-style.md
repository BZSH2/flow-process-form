# 代码样式规范

## 目标

统一团队代码可读性与维护成本，减少样式和结构上的随意性。

## 核心规则

- 使用语义化命名，避免无意义缩写。
- 优先保持函数单一职责，避免超长函数。
- 复用已有工具函数与配置，避免重复实现。
- 变更时尽量保持与同目录已有代码风格一致。
- 优先使用显式类型和类型推导结合的写法。
- 谨慎使用 `any`，优先 `unknown` + 类型收窄。
- 公共数据结构优先放在 `src/types` 或模块内类型定义中。
- 新增常量值优先提取为具名常量，避免魔法值散落。
- 组件命名采用 PascalCase，页面与业务组件职责分离。
- 保持 JSX 结构清晰，避免过深嵌套。
- 样式统一通过 Stylelint + Prettier 检查。

## 落地检查

- `pnpm format`
- `pnpm lint`
- `pnpm typecheck`
